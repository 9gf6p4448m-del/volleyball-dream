- **B1**：中強度。逐出→全隊 `lineup.trust` −5（夾限 ≥0）、每屆上限 1 次；主角不受影響；

# Part A 賽季輪迴（六題）

### A1 保留／重置清單

草稿設計：季末（奪冠或止步）→「進入下一屆」。

| 保留（帶著走） | 重置（新一屆） |
|---|---|
| 主角屬性/技巧/成長點數 | 賽程（同六場模板） |
| 名冊隊友成長 | 戰績（0 勝 0 敗重來） |
| 招募進度與已入隊成員 | 每場比賽種子（決定論衍生、不重演上一屆） |
| 宿敵記憶（scouting 跨屆累積） | |
| 已播劇情（teach-*/debut 不重播） | |
| 先發編排（lineup/trust） | |

- **A.（推薦）照此清單**——「帶著走」的都是玩家資產（成長/招募/記憶/技巧），
  「歸零」的都是賽事程序（賽程/戰績/種子）。
- B. 圈選調整——想改哪一項（例如宿敵記憶也重置、或戰績跨屆累計顯示），圈出來
  逐項議。

### A2 難度模型（Sawmah 已提的顧慮：「止步的人還要打更強的隊，不合理」）

- **A.（推薦，草稿已實作）難度綁成就，不綁屆數**：止步 → 對手**維持原強度**（變強的
  是你：名冊成長＋招募＋技巧——失敗的回報是帶著成長捲土重來）；奪冠 → titles+1 →
  衛冕屆對手全屬性 **+3×titles**（敘事：全國都在研究衛冕軍，與宿敵情蒐呼應）。
- B. 每屆固定微升（不分輸贏）——即 Sawmah 覺得不合理的那種，列出供對照。
- C. 學年制新老交替（畢業/新生、各隊浮動可強可弱）——最真實但敘事/命名工程量大，
  適合當 Phase 4+ 遠期方向。
- 若選 A：衛冕升級幅度（現 +3/冠）可一併拍。

### A3 賽中換人／輪轉替補要不要進 W5？

換人是 FIVB 規則 7.7（輪轉錯誤追溯扣分）驗證器 `cancelFaultPoints` 的真實觸發源
（W3 已把接線點定死在 sim 純註解、零執行碼，就等這個功能）。
- 進 W5：規則完整度高，但工程大（換人 UI＋輪轉驗證啟用＋決定論影響）。
- **留 W6+（推薦）**：W5 已有輪迴＋逐出兩大主題，換人獨立成週。

### A4 新一屆的劇情量級

現況：已播事件不重播＝第二屆零對話直接開打。
- **最小（推薦）**：衛冕開場一段＋捲土重來開場一段（兩段隊長對話；與命名工程一起寫
  也可，先上佔位標 `TODO(naming)`）。
- 中量：另加宿敵跨屆差分（「又是曜石——第三年了」）。
- 零劇情：全部留命名工程。

### A5 招募節奏確認

跨屆累積下的達成屆數：曜石（一屆遇 2 次）＝最快 1 屆；天鷹＝決賽勝 1 屆；白浪/鐵霧
（勝 2）＝至少 2 屆；北原（勝 3）＝至少 3 屆。**全招滿約需 3 屆**。

- **A.（推薦）維持現值**——招募是多屆長線目標，配輪迴的節奏剛好；改快了第 2 屆
  就集滿，Phase 4 前沒得追。
- B. 放寬一檔（北原 3→2、白浪/鐵霧 2→1）——約 2 屆集滿，爽感前移但長線變短。
- C. 其他數字——直接寫在回填處（改 `RECRUIT_CONDS` wins，一行/隊）。

### A6 棄賽 UX（實測踩雷：四場 0:25）

中途離開＝棄賽敗 0:25（拍板的防 reload 機制）。實測：測試/誤觸離開也吃敗場，
一屆四場 0:25 的體驗很差。
- **推薦：離開前確認彈窗**「中途離開將記棄賽敗（0:25）——確定離開？」機制不變。
- 替代：每屆 1 次無代價棄賽額度（機制複雜化，不推薦）。

# Part B 逐出機制（六題，依骨架）

### B1 逐出成本強度

背景：隊友 trust 基準 20、招募新人 10、主角另計（save.player 不受影響）。

- A. 輕：全體 −3、無次數限制。代價感弱，可能被當免費洗名冊。
- **B.（推薦）中：全體 −5、每屆上限 1 次**。−5 有感（20→15 一傳分配肉眼可見變少）
  但不毀隊；上限 1 次/屆把逐出定調成「重大決策」、天然防洗名冊。
  **persona 差分（好友掉更多）本輪不做**——persona 現為一句人設文字、無關係資料
  （誰跟誰好），差分基礎要等命名工程定人設，屆時同一機制上加權即可。
- C. 重：全體 −8＋該屆禁再招募。逐出本為了招人，禁招自相矛盾。

### B2 不可逆的實作邊界

- A. 全不可逆＋條件全清（progress 歸零）：違反「progress 永不重置」原則、沒收玩家
  壯舉進度——體感差。
- **B.（推薦）不可逆但不沒收**：
  - 逐出者本存檔**永不回歸**；`recruited` 標記**保留**（同隊不可再招——防重複入隊
    與 R id 語義混亂）；progress 保留但招募列顯示「已離隊」（凍結非歸零）。
  - **例外（骨架點名）**：「條件已達成但因額滿未入隊」者**不受影響**——逐出騰位後
    照常自動入隊（這正是逐出的目的；快照 §9 鉤子零新接線）。
  - **先發保護**：在 `lineup.starters` 內者不可逐——UI 擋「先把他移出先發」；避免
    逐出瞬間先發缺人、lineup 治癒連鎖重置的邊界地獄。
  - **範圍縮小（呼應前提）：本輪僅可逐 R 成員（招募生），創隊班底不可逐**——班底綁
    劇情/隊長身分/傳授鏈，逐出的敘事後果超出佔位台詞能承載；Phase 4 劇情強化再開放。
- C. 軟可逆（花代價請回）：骨架已排除（自由球員市場→Phase 4），不列。

### B3 確認流程形式

- **A.（推薦）二次點擊**：第一按變紅「將永久失去 {名}——再點一次確認」＋後果一行
  **全揭露**（trust −5、本屆不可再逐、該隊不可再招）。與既有「開始生涯覆蓋確認」
  同範式、手機友善、零新互動範式。
- B. 長按 1.5s：全案皆 pointerdown 離散點按，為單一操作引入新範式，不推薦。
- C. 打字確認（輸入名字）：儀式感可由告別台詞（命名工程）承擔，手機輸入負擔大。

### B4 expelled 結構——另立，不塞 scouting

`career.scouting` 鍵是**對手隊 id**、值是「該隊看過我什麼」（情報語義）；逐出的是
**我方成員**——共用＝語義錯位（且 origin='starter' 者無對手鍵可掛）。另立兩案：

- **A.（推薦）`save.recruitment.expelled: []`**——逐出是招募的逆操作，同域同處：
  schema 驗證同段、applyRecruit/applyExpel 成對、RMW 同鍵原子性天然成立。
- B. `save.story.expelled`——語義也通（Phase 4 劇情素材），但 story 是 Phase 4 預留
  空物件（W1 拍板「不要猜」），現在定型吃掉 Phase 4 結構自由度。
- 條目欄位（Phase 4 回歸對戰所需一次存齊）：

```js
{ member: {完整成員快照},   // id/name/origin/role/attributes/growth/dna/height/persona（同 W4 成員形狀）
  seasonIndex,             // 第幾屆逐出（回歸時算成長年數）
  titlesAtExpel }          // 逐出當時冠軍數（回歸強度基準；決定論、無時間戳）
```

### B5 佔位台詞規格

- A. 零台詞：逐出是情感重量最大的操作，全無反應很出戲。
- **B.（推薦）2 行極簡**：被逐者一行＋隊長一行，語氣平靜克制（「…知道了。」／
  「我會跟大家說的。」量級）——正式告別戲留命名工程（名字/persona 未定寫不出好台詞）。
- C. 4–6 行小劇：沒有人設寫不好，會重寫兩次。
- **標記規格（採納骨架建議）**：統一 `// TODO(naming)`；台詞常數集中 `EXPEL_LINES`
  （events.js，與對話系統同域）。
- **附帶（零行為改動）**：本輪順手把**既有**佔位點全補 `// TODO(naming)`——
  STARTER_DEFS 名字/persona（roster.js）、RECRUIT_DEFS（recruitment.js）、events.js
  角色代號 speaker、opponents.js「北原工商1號」——命名工程一條
  `grep -rn "TODO(naming)" src/` 撈齊，不再漏。

### B6 測試範圍（最小集合）

性質＝驗收清單確認題：**A.（推薦）照下列六項全收**；B. 圈選增刪（想加的情境或
認為過度的項目，圈出來議）。

1. **applyExpel 原子性**：單次 RMW 同時完成 members 移除＋lineup.trust 全體下修＋
   expelled 記錄——無「移了人沒扣 trust」中間態。
2. **trust 夾限**：全體 −5 不低於 0；主角（save.player）不受影響。
3. **邊界擋下**：先發中不可逐；每屆限 1 次；創隊班底不可逐。
4. **招募連動（核心閘）**：額滿＋條件已達成 → 逐出 → 回畫面自動入隊（§9 鉤子實測）；
   被逐隊不重複入隊；R id 不回收。
5. **schema**：expelled roundtrip；舊檔（無 expelled 欄）升級冪等不 brick。
6. **回歸**：未逐出存檔行為 byte-identical。

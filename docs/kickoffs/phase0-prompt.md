你要為一個叫「排球夢」的3D排球生涯遊戲建立專案骨架，並完成 Phase 0（基準測試）。這是一個 PC + 手機雙平台的 PWA，技術棧 Three.js。

## 開始前
1. 讀 `docs/ROADMAP.md`——這次只做 **Phase 0**，不要碰 Phase 1 以後的任何內容。
2. 讀 `docs/design-brief.md`——這是背景脈絡，用來理解「架構鐵律」。你現在蓋的是一個未來要長成完整生涯遊戲（含多人連線）的地基，不是用完即丟的 demo。特別遵守：**模擬核心與畫面/輸入分離**、**可包殼上架的乾淨結構**。即使 Phase 0 用不到，專案骨架的分層要為此預留。

## Phase 0 任務：基準測試場景
目的是在真機上判斷「寫實比例美術」是否可行。做出一個可部署的場景，包含：
- 一座標準排球場（正確尺寸比例、球網、界線）
- **12 個寫實比例的人形模型，每個都帶動畫**（待機/移動動畫即可；用 Mixamo 系動捕動畫或等效免費資源）
- 一顆有物理的排球（重力、彈跳、與地面/網的碰撞）
- 畫面上即時顯示 **FPS 數字**與當前 render 更新率

## 硬性要求
- **模擬固定步長**：物理/邏輯以固定頻率運算，與 render 幀率脫鉤。60fps 和 120fps 下行為必須一致。
- **不鎖幀**：requestAnimationFrame 自然跟隨裝置更新率，不要人工鎖 60。
- **手機優先**：觸控與滑鼠都能操作鏡頭（這階段有基本 orbit 檢視即可）。
- **禁止自我降級特效**來硬湊幀數——我們要的是真實的效能數字。
- PWA 基本設定（manifest + service worker），能在手機瀏覽器加到主畫面。

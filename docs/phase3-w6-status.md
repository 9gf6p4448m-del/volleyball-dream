## 2. 結構要點（W7+ 接手必讀）

- **recruitKey 鍵結構**：`RECRUIT_CONDS/DEFS` 改 `recruitKey → {opponentId, role, wins?, feat?, stage?}`。
  既有 5 隊 recruitKey＝opponentId（舊存檔 progress/recruited/生成 hash 零遷移、重演逐值一致）；
  新增鍵 `{oppId}-2`。成員新欄 `member.recruitKey`（同隊多人 origin 不再唯一；舊成員缺欄回退 origin）。
  `origin`/`dna.teamId` 維持「來源隊」語義（舊隊情結／逐出顯示靠它）。
- **賽程輪抽**（`schedule.js`）：`drawGroupOpponents({seed, invitedId, prevGroupIds})`——
  保底債（上屆小組缺席的僅小組隊必入）＋每屆 ≥min(2,僅小組隊數) 席留給僅小組隊＝
  「每隊 ≤2 屆必現」不變式（對抗情境測試：連續 12 屆邀天鷹仍成立）。
  第 1 屆恆為故事模板（見 §3 偏差 1）。schedule 存檔即公開（invited 旗標在賽程項上，零 schema 手術）。
- **sim 換人**：`createGame({benches})` → `state.bench/subs/serveSeq/rotationFault`；
  `applySubstitution(state, {team, outId, inId}) → {ok, reason}`＝唯一寫入路徑
  （純 state 副作用不經 Intent，比照 applyLiberoSwaps；發球前快照已含換人後陣容＝VCR 決定論不破）。
  板凳不在輪轉＝tryBlock/AI（全走輪轉名單）零污染——**帶板凳不換人＝事件流逐位一致**（測試把關）。
  新事件型別：`SUBSTITUTION`／`ROTATION_FAULT`／`ROTATION_ADJUST`。
- **帶子鏡像鐵律**（本輪踩坑）：情蒐帶預生成 game 必須與正賽同陣容（含 benches）——
  matchView units 依正賽 players 建，帶子重播 state 缺 actors 會逐幀炸（見 §4 bug 2）。
- **調參點**：`TUNING.SUBS_PER_SET`（6）；招募條件數值在 `RECRUIT_CONDS`；
  微指標門檻（接發 0.8）在 `subPanel.js statLine`。

## 4. 本輪抓到並修復的 bug（皆有回歸測試）

1. **W5 潛伏：跨屆隊友成長全滅**——`applyRosterGrowth` 冪等鍵只認 matchId，賽季輪迴後
   matchId 每屆重複（group-1…）→ 第二屆起全部隊友成長被誤判「已長過」靜默跳過。
   修＝log 條目帶 season 欄（舊條目回退第 1 屆；season===1 不寫欄位＝第 1 屆序列化逐位不變）；
   settleCareerMatch 經 `store.seasonIndex()` 傳屆數。**跨屆治具第一次跑就抓到——C1 的存在價值當場兌現**。
2. **scoutTape 板凳鏡像**：帶子預生成 game 未帶 benches → matchView（16 units）在帶子
   重播 state 查無板凳 actors、每幀炸 `divedUntil of undefined`（Playwright 實跑抓到，
   node 測試抓不到的表現層整合缺口）。修＝`buildScoutTape` 收 benches 與正賽同陣容。
